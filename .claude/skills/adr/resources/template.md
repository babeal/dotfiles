# ADR Template

Use this exact structure:

```markdown
# {number | short}. {title | sentence-case}

Date: {YYYY-MM-DD}

## Status

{Proposed, Accepted | Superseded by [ADR NNN](NNN-name.md) | Deprecated}

## Context

{Explain the problem space and why a decision is needed. Be conversational and practical. If there are multiple options worth listing, describe them briefly — include links to tools/libraries on first mention. Reference project constraints (e.g., prototype scope, time, requirements) when relevant.}

## Decision

{State what was chosen and why in 1-2 concise paragraphs. Be direct. Reference the specific tool/library/approach chosen. It's fine to use first person when sharing personal reasoning or experience.}

## Consequences

{Describe trade-offs honestly. Can be prose paragraphs or structured with Positive/Negative bullet lists when there are many points. Mention what would trigger revisiting the decision. Keep it practical — focus on what matters for the team.}
```
